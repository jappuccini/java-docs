## Klassendiagramm

```mermaid
classDiagram
    Vehicle <|-- Car : extends
    Vehicle <|-- Truck : extends
    Vehicle o-- Engine
    Rental o-- Vehicle
    Partner <|.. Rental : implements
    TravelAgency o-- Partner

    class Vehicle {
        <<abstract>>
        -make: String
        -model: String
        -engine: Engine
        #speedInKmh: double
        -numberOfVehicles: int$

        +Vehicle(make: String, model: String, engine: Engine)
        +getMake() String
        +getModel() String
        +getEngine() Engine
        +getSpeedInKmh() double
        +accelerate(valueInKmh: int) void &#123final&#125
        +brake(valueInKmh: int) void &#123final&#125
        +toString() String &#123abstract&#125
        +getNumberOfVehicles()$ int
    }

    class Engine {
        <<enumeration>>
        DIESEL = Diesel
        PETROL = Benzin
        GAS = Gas
        ELECTRO = Elektro
        -description: String &#123final&#125
        +getDescription() String
    }

        <<final>>
        -seats: int &#123final&#125

        +getSeats() int

        +toString() String
    }

    class Truck {
        <<final>>
        -cargo: int &#123final&#125

    class Rental {
        -name: String &#123final&#125

        +transformAllTrucks() void

    }

    class Partner {
        <<interface>>
        +toString() String
    }

    class TravelAgency {
        -name: String
        -partners: ArrayList~Partner~
        +TravelAgency(name: String)
        +addPartner(partner: Partner) void
        +toString() String
    }
```

## Hinweise zur Klasse _TravelAgency_

- Der Konstruktor soll alle Attribute initialisieren
- Die Methode `void addPartner(partner: Partner)` soll dem Reisebüro einen
  Partner hinzufügen
